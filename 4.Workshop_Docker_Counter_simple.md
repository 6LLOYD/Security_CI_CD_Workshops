# Utilisation de Dockerfile dans une intégration continue (et en local)

## Ajout d'un Dockerfile (on y reviendra, pour l'instant, trust)
- Ajouter un fichier "Dockerfile" sans extension (et sans les guillemets) à la racine avec le contenu suivant
```Dockerfile
# On défini la base du conteneur
[A remplir]
# On défini le répertoire de travail dans le conteneur
[A remplir]
# On copy tout le contenu (normalement, on devrait retirer les dossier générés comme __pycache__ et .pytest_cache)
[A remplir]
# On build le projet dans le conteneur
[A remplir]
# On lance la commande python app.py (pour lancer l'appli)
[A remplir]
```
- Pour vérifier que le conteneur marche, à la racine dans votre terminal :
```sh 
docker build -t mon-image-python .
docker run mon-image-python
```

## Ajouter un scan du conteneur avec Trivy (sécurité)
- Dans le fichier yaml contenant la CI que vous avez créée, ajouter un scan Trivy.
- C'est un peu plus complexe, alors je vous le donne. Il faut d'abord builder l'image docker via le Dockerfile
- Puis démarrer le scan trivy sur cette image
```yaml
    - name: Build Docker image
      run: |
        docker build -t devops-demo .

    - name: Scan Docker image with Trivy
      uses: aquasecurity/trivy-action@master
      with:
        image-ref: devops-demo
```
