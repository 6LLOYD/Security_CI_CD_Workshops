# Workshop pour la réalisation d'une intégration continue (Python)

## Création d'un nouveau repository git (github)
- Créer un nouveau repository github
    - Pensez à créer un intégrer un README.md (même s'il est vide - Pas de description)
    - Pensez ABSOLUMENT à mettre un .gitignore, vous pouvez utiliser celui proposé par github à la création du repo (pour Python)
    - Finaliser la création du repo
- Clonez le repo dans un dossier vide en local
```git clone {...VOTRE_URL...} .```
- N'OUBLIEZ PAS LE . à la fin de la commande

## Phase de "code"
- Ajoutez un fichier app.py (à la racine du dossier où l'on a fait notre clone):
```py
from counter import add_one_to_count, remove_one_to_count

def main(): 
    count = add_one_to_count(0)
    print('Affichage du compteur après ajout : ' + str(count))
    count = remove_one_to_count(1)
    print('Affichage du compteur après retrait : ' + str(count))

main()

```
- Ajoutez un fichier counter.py (au même niveau, donc à la racine) :
```py
def add_one_to_count(count):
    count = count + 1
    return count

def remove_one_to_count(count): 
    if (count > 0):
        count = count - 1
    return count

```

## Ajouter le fichier contenant les dépendances
- Créer un fichier requirements.txt (à la racine aussi) contenant les dépendances :
```
pylint
bandit
pip-audit
```
- Buildez les dépendances de votre projet (depuis le terminal, au même niveau que le fichier requirements.txt) dans un terminal :
```sh
pip install -r requirements.txt
```

## Faites une analyse syntaxique avec pylint
- En ligne de commande (toujours à la racine) :
```pylint *.py```
- Normalement, vous allez avoir des erreurs !
- Ajoutez un fichier .pylintrc à la racine également :
```txt
[MESSAGES CONTROL]
disable=
    C0114,
    C0116
```
- Le but de ce fichier est de désactiver les remontées un peu trop contraigantes de pylint (en général, on commente pas chacune des méthodes)
- Corriger les quelques remontées de pylint, s'il y en a encore (sûrement des sauts de ligne à la fin de vos fichiers ou des espaces en trop en fin de ligne...)

## Faites une analyse de sécurité avec bandit
- En ligne de commande (toujours à la racine) :
```bandit -r .```
- Normalement, avec un code aussi basique, vous n'aurez aucune remontée d'erreurs

## Faites un scan des dépendances avec pip-audit
- En ligne de commande (toujours à la racine) :
```pip-audit```
- Normalement, avec un code aussi basique, vous n'aurez aucune remontée d'erreurs
- En vrai, y'aura peut-être des erreurs, en fonction de votre version de python, mais dans la CI, on en aura sûrement pas avec la v3.11 de python

## Créer une CI complète
- Le but de cette CI est de lancer
    - Build (build des dépendances)
    - Pylint (analyse syntaxique)
    - Pip-audit (dépendances)
    - Bandit (analyse sécurité)
- Je vous laisse un peu chercher sur ça, mais vous avez tout ce qu'il faut :
    - En démarrant d'un workflow de CI depuis github actions
    - En ajoutant des steps qui correspondent aux étapes que vous avez faites en local précédemment
- Pensez à bien faire votre add / commit / push (ou via pull request si vous avez bien assimilé git et les branches) pour lancer le workflow d'intégration continue
- Vérifier que tout se passe correctement (depuis l'onglet actions dans github)
- Si des erreurs persistent, tentez de les corriger ou demandez-moi si vous n'y arrivez pas.
- Vous avez fait votre première CI !!!
