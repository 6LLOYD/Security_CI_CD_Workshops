## Créer un endpoint asynchrone en python
- On va ajouter un autre fichier main.py (ou remplacer l'existant) par ce contenu : 
```py
import os
from datetime import datetime, timezone

import redis.asyncio as redis
from fastapi import FastAPI

REDIS_HOST = os.getenv("REDIS_HOST", "redis")
REDIS_PORT = int(os.getenv("REDIS_PORT", "6379"))
REDIS_DB = int(os.getenv("REDIS_DB", "0"))
REDIS_PASSWORD = os.getenv("REDIS_PASSWORD")

r = redis.Redis(host=REDIS_HOST, port=REDIS_PORT, db=REDIS_DB, password=REDIS_PASSWORD)

app = FastAPI(title="Simple API (async)")

@app.get("/health")
async def health():
    return {"status": "ok"}

@app.post("/stress")
async def stress(count: int = 1000, ttl: int = 60):
    value = "Osef de la valeur"
    ts = datetime.now(timezone.utc).strftime("%Y%m%d-%H%M%S-%fZ")

    pipe = r.pipeline(transaction=False)
    for i in range(count):
        key = f"stress:{ts}:{i}"
        pipe.set(name=key, value=value, ex=ttl)
    await pipe.execute()

    return {"created": count, "ttl_seconds": ttl, "key_prefix": f"stress:{ts}:"}

if __name__ == "__main__":
    import uvicorn
    port = int(os.getenv("PORT", "8000"))
    uvicorn.run("main:app", host="0.0.0.0", port=port)
```
- Pour l'explication rapide, on ajoute redis pour une base de données qui exploite la RAM, ce qui permettra de voir facilement les variations lors du monitoring.
- On a deux endpoints, un pour vérifier que l'API est up (/health) et un pour faire stresser la RAM (/stress)
- On ajoute plusieurs clés dans la RAM ce qui va stocker des données dans la RAM et donc la faire grimper. Avec un timeout à 60 secondes pour éviter de la surcharger indéfiniment.
- On doit adapter le requirements.txt :
```
pytest
pylint
bandit
pip-audit
psycopg2-binary
fastapi
uvicorn[standard]
redis
```
- uvicorn, c'est un package qui permet de mieux gérer notre API et fastapi, c'est le package qui nous permet de créer une API en python simplement.

## Adapter le Dockerfile
- Normalement, rien à changer ou presque, c'est juste que jusqu'à maintenant normalement on était sur un fichier python app.py et on veut passer sur main.py comme fichier principal

## Ajout des métriques que l'on veut scraper avec prometheus
- Créer un dossier "prometheus" à la racine du projet
- Ajouter un fichier prometheus.yaml
```yaml
global:
  scrape_interval: 1s

scrape_configs:
  - job_name: "redis"
    static_configs:
      - targets: ["redis-exporter:9121"]

  - job_name: "prometheus"
    static_configs:
      - targets: ["prometheus:9090"]
```

- Dans le dossier provisioning, ajouter deux sous dossiers :
    - dashboards
    - datasources
- Dans dashboards, ajouter un fichier dashboard.yml, celui ci permet de créer un dossier contenant nos futurs dashboards :
```yaml
apiVersion: 1

providers:
  - name: default
    orgId: 1
    type: file
    folder: Redis
    allowUiUpdates: true
    disableDeletion: false
    updateIntervalSeconds: 10
    options:
      path: /var/lib/grafana/dashboards
      foldersFromFilesStructure: false
```
- Dans le dossier datasources, ajouter un fichier datasource.yml, celui-ci permet de configurer la datasource exploitée dans les dashboards (donc les métriques de redis) :
```yaml
apiVersion: 1

datasources:
  - name: Prometheus
    type: prometheus
    access: proxy
    url: http://prometheus:9090
    isDefault: true
    editable: false
```

## Appeler votre API et voir le monitoring
- Soit vous faites du CURL
- Ou alors vous utilisez un outil comme postman pour appeler :
    - En get : http://localhost:8000/health
    - En post (pas besoin de body) : http://localhost:8000/stress
- Après un appel à http://localhost:8000/stress vous pouvez voir votre monitoring fonctionner sur prometheus à l'adresse suivante : http://localhost:9090/
- Vous recherchez la metrique "redis_memory_used_bytes"
- Vous cliquez sur l'onglet "Graph"
- Vous cliquez sur "Execute" en haut à droite et vous verrez les données en "temps réel" (toutes les secondes, ça scrap)
- Pour Grafana, l'url est : http://localhost:3000/login
    - Login : admin
    - Password : admin
- Skippez le changement de mot de passe
- Dans l'onglet "home" vous avez déjà un premier dashboard
- Dans l'onglet "dashboards", puis dossier "redis", vous avez deux autres dashboards.
- A partir d'ici, il est possible de créer des alertes, normalement, j'ai du vous montrer comment faire, mais c'est une simple boite à cliques :
    - On créer une alerte, on rempli les différents champs, les éléments qui n'existent pas on met des choses par défaut
    - Le seul vrai élément important, c'est de choisir "redis_memory_used_bytes" dans la query que l'on veut faire
    - Régler les paramètres pour lever des alertes (exemple, vérification toutes les secondes etc...)
    - Et créer un contact endpoint. On fait souvent : par email ou par webhook. 
    - Moi j'ai du vous montrer avec discord parce que c'est plus fun
